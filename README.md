# 自費療程月報統計系統

上傳含多月分頁的 Excel，自動統計每月新客人數、費用與療程施作次數。

## 快速啟動

```bash
pip install -r requirements.txt
python app/app.py
```

瀏覽器開啟 http://localhost:5000

## 使用流程

1. 上傳 Excel（每個 Sheet = 一個月）
2. 選擇要分析的月份、設定欄位對應（新客欄、費用欄、療程欄）
3. 點「開始分析」查看統計結果

# Google 商家評論回覆產生器

單一 HTML 檔案（`content_generator/index.html`），**不呼叫任何外部 API，不需要任何金鑰**，因此可以直接部署到 GitHub Pages 等純靜態託管。

輸入品牌名稱、醫師名稱（選填）、主關鍵字、相關關鍵字、文風、特色優勢、服務數據，直接產生一段約 200 字、適合回覆 Google 商家評論、也容易被 AI 引擎摘錄引用的內容。

- 段落順序是**固定的敘事邏輯**（開場 → 回應客戶評論內容 → 特色優勢 → 服務數據 → 相關服務 → 醫師 → 結尾），每個位置的句子仍會隨機挑選不同說法，「換一批」會得到不同措辭，但不會把句子順序打亂成不連貫的組合
- 服務數據（選填，例如「10年經驗, 500+案例, 98%滿意度」）會套上「具體數字 + 結論句 + 來源引導句」的格式（例如「根據我們 10 年經驗、500+ 案例的服務經驗，...這也是我們持續努力的方向」），呼應 GEO 分析中「段落內容要有數字佐證」的引用規則
- 品牌名稱只在開場與結尾各出現一次，中間句子改用「我們」，避免整段每句都塞品牌名稱、讀起來像洗版行銷文案
- 純模板拼接（多種文風各有多組句型），**不使用任何 AI/LLM**，因此可以做到零金鑰、零後端；但也因此無法真正理解評論語意，只能做關鍵字層級的比對與引用
- 若有填寫醫師名稱，產出內容一定會提及；若留空則完全不會出現醫師相關句子
- 可貼上客戶評論內容：工具會用**關鍵字比對**（不是語意理解）偵測評論提到態度／環境／專業／效果等重點，直接引用一小段客戶寫的話並附上對應的回應句，緊接在開場之後回應，讓回覆更貼合這則評論；如果評論鼓勵其他人考慮，也會隨機補上一句「這份分享能幫助猶豫中的朋友」
- 引用片段會優先挑「有提到具體重點、長度適中」的句子，而不是死板地抓第一句硬切——避免抓到鋪陳/抱怨的背景描述，或把句子從中間切斷
- 若評論內容偵測到負面關鍵字（如「態度不好」「很失望」「不推薦」），會顯示提醒，且引用句會改用道歉/檢討語氣，而不是套用感謝用語；仍建議由專人親自回覆負評，不要直接套用
- 依文風隨機附上合適的表情符號（親切溫暖／活潑熱情較多，專業信賴較少，簡潔俐落不使用）；活潑熱情風格的結尾避免用「下次再來玩」「不見不散」這類把回診/療程講得像約出去玩的說法，對醫療業不夠尊重
- 內容為通用草稿，發布前建議依實際情況微調用詞

## 使用方式

直接用瀏覽器開啟 `content_generator/index.html`，或部署到 GitHub Pages 後開啟對應網址。

1. 輸入品牌名稱、主關鍵字（必填），醫師名稱、相關關鍵字、特色優勢、服務數據與客戶評論內容建議填寫
2. 選擇文風（專業信賴／親切溫暖／活潑熱情／簡潔俐落）
3. 點「產生內容」，不滿意可點「換一批」重新隨機組合
4. 複製內容，貼到 Google 商家評論回覆前建議再讀一次微調（若出現負評提醒，請改由專人回覆）
