# GEO 品牌引用分析器

分析「GEO 追蹤工具」（AIO / AI Mode / Gemini / GPT 回應紀錄）匯出的 CSV，找出 AI 到底
根據什麼邏輯提及/引用一個品牌，並把這個邏輯轉成可以套用在其他關鍵字、其他網站上的
內容撰寫建議。

## 這份分析想回答的問題

1. 品牌在 AI 回應中被提及/引用的真實程度是多少？
2. AI 引用的來源都是誰（自家網站 / 社群 / 政府衛教 / 醫學文獻 / 競品）？
3. 自家網站裡，哪些內容已經被引用，這些內容有什麼共同特徵？
4. 什麼因素會讓引用率大幅提升？
5. 把 1-4 的答案變成下一篇 SEO 文章的具體寫作規則。

## 資料來源（GEO 匯出的 6 種 CSV）

放在 `data/raw/`（此目錄已被 `.gitignore` 排除，不會進版控 — 這些是實際業務/回應資料）：

| 檔名關鍵字 | 內容 |
|---|---|
| `GEO_ResponseList` | 逐筆 AI 回應紀錄：提示詞、完整回應文字、是否提及自家/競品、引用網域等 |
| `01_Overview_Metrics` | 整體期間的品牌提及率、網站引用率、情緒分數、聲量佔比、平均排名 |
| `02_PromptAnalysis` | 依主題/關鍵字/提示詞拆解上述指標 |
| `03_CompetitiveAnalysis` | 品牌 vs 競品的提及率/引用率/聲量佔比排行 |
| `04_Citation_Domain` | 被引用的網域清單與次數、是否為自家網域 |
| `05_Citation_URL` | 被引用的實際網址清單與次數、是否為自家網址 |

`src/geo_analyzer/loaders.py` 會用檔名關鍵字自動辨識這 6 種檔案，欄位一律轉成英文
內部欄名，所以之後即使是別的品牌、別的期間匯出的資料，也能直接丟進來跑。

## 目前資料跑出來的關鍵發現

實際針對「月易」品牌 2026-04-21 ~ 2026-07-19 的匯出資料分析後（完整報告見
`reports/findings.md`，由 `cli.py analyze` 產生）：

1. **品牌比對邏輯本身有 bug，嚴重低估提及率。** GEO 工具用品牌簡稱「月易」做精確比對，
   但 AI 幾乎都用全稱「月易診所」回答。工具因此把 354 筆真正在講自己品牌的回應，
   誤判成「競品提及」，還把自己的全稱列成排名第一的競品。加上別名比對後，
   真實提及率從回報的 0.6% 校正為 22.5%。**這是目前最值得先修正的一件事**：
   先確認 GEO 工具的「自家品牌」清單有沒有涵蓋所有慣用稱呼（全名、簡稱、英文名、網域）。
2. **AI 引用的來源有八成是長尾小網域**，其餘依序是社群平台（6.2%）、醫學文獻
   PubMed/PMC（5.9%）、政府衛教網站（4.4%）、自家網站（3.2%）。沒有單一來源能壟斷
   引用——可操作的空間在於成為那個長尾裡「最具體、最直接回答問題」的來源，而不是
   想贏過 PubMed 或政府網站的權威度。
3. **提示詞裡有沒有直接點名品牌，引用率差 6 倍。** 提示詞點名品牌時，平均網站引用率
   58.0%；沒點名時只有 9.5%。代表品牌實體與主題的關聯強度，是目前最大的槓桿——
   要讓 AI 在「未點名品牌」的一般提問中也聯想到你，需要在全網（自家網站、社群、
   第三方報導）持續把「品牌 x 特定主題」的關聯做強。
4. **自家網站被引用最多的頁面，標題都是「痛點情境句 + 具體專業詞」**，例如
   「讓每一次就診都不再是壓力，LGBTQ性別友善門診」「別讓不便決定你的日常，哪些狀況
   適合做肛門整形」，而不是關鍵字堆砌型標題。
5. **主題屬性決定該不該搭配外部權威來源**：醫療風險/技術類主題，AI 傾向同時引用
   政府衛教或醫學文獻佐證；體驗/隱私/友善類主題，品牌第一人稱敘述本身引用率就高，
   不需要額外背書。

## 網頁版（免安裝，直接用）

`web/index.html` 是同一套邏輯（品牌別名校正、網域分類、內容簡報產生）的瀏覽器版本，
不需要 Python，用瀏覽器直接開啟即可：拖放 6 個 GEO 匯出 CSV 進去就會自動跑分析，
另一個分頁可以直接產生內容簡報。所有運算都在瀏覽器本機完成，檔案不會被上傳到任何地方。

## 安裝（CLI 版）

```bash
cd geo-analyzer
pip install -r requirements.txt
```

## 用法

### 1. 分析一批 GEO 匯出資料

```bash
python cli.py analyze \
  --data-dir data/raw \
  --brand 月易 \
  --aliases 月易診所,boweleasy \
  --own-domain boweleasy.com \
  --out reports/findings.md
```

`--aliases` 和 `--own-domain` 是這個工具最重要的輸入——把品牌所有已知的稱呼方式和
網域都列進去，才能正確校正品牌比對邏輯（見上面發現 1）。

### 2. 針對下一篇文章的主題/關鍵字產生 GEO 內容簡報

```bash
python cli.py brief \
  --topic "隱私、美學與性別友善" \
  --keyword "性別友善" \
  --brand 月易 \
  --aliases 月易診所,boweleasy \
  --data-dir data/raw \
  --out reports/brief.md
```

輸出的簡報包含：標題建議、內文結構檢查表、品牌實體強化檢查表、依主題屬性判斷的
權威來源搭配建議，以及（若有提供 `--data-dir`）該關鍵字目前的引用率基準值。

`--data-dir` 是選填的——沒有既有數據時，`brief` 一樣能純粹依規則產生建議，
方便套用在還沒開始追蹤 GEO 數據的新網站/新專案上。

## 套用到其他網站/專案

這個工具刻意把「資料載入」「品牌比對」「網域分類」「內容訊號」「簡報產生」拆成
獨立模組（見 `src/geo_analyzer/`），對應到之後要做的事：

- 換一個品牌/網站：只要換 `--brand` / `--aliases` / `--own-domain`，`domain_classifier.py`
  的社群/政府/醫學文獻分類規則是通用的，不用改。
- 換一個沒有 GEO 追蹤工具、只有自己網站內容的專案：可以直接呼叫
  `content_signals.analyze_title()` / `analyze_response_text()` 分析現有文章，
  再用 `brief_generator.generate_brief()` 產生新文章的寫作規則。
- 之後如果要串接「自動生成 SEO 文章」，`ContentBrief.to_markdown()` 的內容就是
  可以直接丟給文章生成流程的 prompt/checklist。

## 專案結構

```
geo-analyzer/
├── cli.py                     # 指令列入口：analyze / brief
├── src/geo_analyzer/
│   ├── loaders.py             # 6 種 CSV 的載入與欄位正規化
│   ├── brand_match.py         # 別名比對，修正自家/競品誤判
│   ├── domain_classifier.py   # 網域分類（自家/社群/政府/醫學文獻/長尾…）
│   ├── content_signals.py     # 標題/回應文字的結構訊號（提問句、痛點框架、條列…）
│   ├── metrics.py             # 把上述模組組合成各項指標
│   ├── report.py              # 產出完整分析報告（markdown）
│   └── brief_generator.py     # 產出單一主題/關鍵字的內容簡報（markdown）
├── data/raw/                   # 放 GEO 匯出 CSV（已 gitignore，需自行放入）
└── reports/                    # analyze / brief 的輸出（實際跑出的結果可自行留存）
```
